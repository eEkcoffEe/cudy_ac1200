# Документирование процесса расширения памяти (Extroot) на Cudy AC1200
 
**Платформа:** Роутер Cudy AC1200  
**ОС:** OpenWrt 25.12.5 (архитектура пакетов: `apk`)  
**Цель:** Перенос раздела `/overlay` на внешнюю USB-флешку для обхода лимита встроенной памяти (9.3 MB).
 
---
 
## Хронология решения проблем
 
В процессе настройки мы столкнулись с несколькими нюансами операционной системы, которые успешно исправили:
 
1. **Отсутствие поддержки файловых систем Microsoft:** Изначально флешка была отформатирована в FAT32/exFAT. Для Extroot в Linux требуются права доступа файлов, которые эти системы не поддерживают. Решено форматированием диска в `ext4`.
2. **Отсутствие диска в системе (`ls /dev/sd*` возвращал ошибку):** Роутер видел USB-устройство физически, но не определял его как накопитель. Решено установкой пакета `kmod-usb-storage`.
3. **Ошибка `mount: Invalid argument`:** Попытка примонтировать отформатированную флешку завершалась ошибкой, так как в ядре новой сборки отсутствовал модуль для чтения Ext4. Решено установкой `kmod-fs-ext4`.
4. **Специфика менеджера пакетов:** В данной версии OpenWrt вместо устаревшего `opkg` используется новый менеджер `apk` (команды вида `apk add`).
 
---
 
## Пошаговый ручной процесс выполнения
 
Для истории, все команды, которые привели к успешному результату:
 
# 1. Установка пакетов для работы с USB и файловой системой ext4
apk update
apk add kmod-usb-storage kmod-usb-storage-uas block-mount kmod-fs-ext4 e2fsprogs tar
 
# 2. Перезагрузка для инициализации ядра
reboot
 
# 3. Форматирование накопителя прямо на роутере
mkfs.ext4 /dev/sda1
 
# 4. Создание структуры fstab
block detect > /etc/config/fstab
 
# 5. Перенос структуры overlay на флешку через tar-пайп
mkdir -p /mnt/sda1
mount /dev/sda1 /mnt/sda1
tar -C /overlay -cvf - . | tar -C /mnt/sda1 -xf -
umount /mnt/sda1
 
# 6. Переключение оверлея на флешку с помощью UCI
uci set fstab.@mount[0].target='/overlay'
uci set fstab.@mount[0].enabled='1'
uci commit fstab
 
# 7. Финальный перезапуск устройства
reboot

# Инструкция по установке OpenWrt на Cudy TR1200 v1

Краткое руководство по прошивке роутера Cudy TR1200 v1 с заводского ПО (OEM) на чистый OpenWrt через промежуточный образ.

---

## ⚠️ Важное примечание
* Роутер Cudy TR1200 v1 официально поддерживается только в стабильных версиях OpenWrt, начиная с **23.05.4** и новее.
* Графический веб-интерфейс (LuCI) встроен **только в стабильные релизы (Stable)**. В сборках типа *SNAPSHOT* веб-интерфейс по умолчанию отсутствует.

---

## Пошаговая инструкция

### Шаг 1. Подготовка промежуточной прошивки
Заводская прошивка принимает только образы, подписанные RSA-ключом. Для обхода этого ограничения используется промежуточный переходный образ OpenWrt.

1. Скачайте архив с промежуточной прошивкой по официальной ссылке: [Cudy OpenWrt Software Download](https://www.cudy.com/blogs/faq/openwrt-software-download) (перенаправляет на [Google Диск]([https://www.cudy.com/blogs/faq/openwrt-software-download](https://drive.google.com/drive/folders/1BKVarlwlNxf7uJUtRhuMGUqeCa5KpMnj)). 
2. Найдите и скачайте архив с именем **`TR1200 V1.zip`**.
3. Распакуйте архив и извлеките файл **`TR1200-OpenWRT-Flash.bin`**.

### Шаг 2. Прошивка промежуточного образа
1. Подключитесь к роутеру и перейдите в веб-интерфейс: [http://192.168.10.1/](http://192.168.10.1/)
2. Введите пароль по умолчанию: `admin`
3. Перейдите в раздел **Обновление прошивки (Upgrade Firmware)**.
4. Выберите распакованный файл `TR1200-OpenWRT-Flash.bin` и запустите обновление.
5. **Подождите около 2 минут**, пока устройство перезагрузится.

### Шаг 3. Установка финальной версии OpenWrt
После перезагрузки на роутере будет работать промежуточная система. 

> ❗ **Внимание:** На этом этапе Wi-Fi отключен по умолчанию. Подключите компьютер к роутеру с помощью сетевого кабеля (в любой LAN-порт).

1. Откройте браузер и зайдите в интерфейс LuCI: [http://192.168.1.1/](http://192.168.1.1/)
2. Перейдите на сайт [OpenWrt Firmware Selector](https://firmware-selector.openwrt.org/).
3. Введите в поиск `Cudy TR1200 v1`, выберите стабильную версию (не ниже 23.05.4) и скачайте файл с типом **`sysupgrade`**.
4. В веб-интерфейсе роутера перейдите в меню: **System** -> **Flash Firmware**.
5. Загрузите скачанный файл `sysupgrade` и подтвердите прошивку.

После перезагрузки роутер будет готов к работе на финальной стабильной версии OpenWrt.
